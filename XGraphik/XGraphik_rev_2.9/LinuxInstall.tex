\section{Vorbereitungen für die Installation}

In dieser Anleitung wird die Installation der speziell für das Programmierpraktikum
im Sommersemester 1994 erstellten LINUX-Distribution beschrieben. Dabei wird als Beispiel
ein AT-kompatibler PC mit 80486 Prozessor, Super-VGA-Grafikkarte und AT-Bus-Festplatte
benutzt. Die Festplatte ist vor der Installation vollständig mit MS-DOS belegt und soll
nach der Installation sowohl LINUX als auch MS-DOS enthalten.

Machen Sie deshalb zuerst ein Backup der MS-DOS-Daten und -Programme,
die Sie behalten möchten. Dazu
können Sie entweder einzelne Files auf Diskette kopieren oder den Inhalt der ganzen
Platte mittels des MS-DOS-Befehls \prog{backup} auf Disketten kopieren.
Nachdem LINUX installiert ist, können Sie mit dem Programm
\prog{restore} das Backup wieder auf die Festplatte spielen.
Weitere nützliche Programme zum Sichern und Wiedereinlesen von Daten sind
\prog{copy} und \prog{xcopy}.

Halten Sie Ihre orginalen MS-DOS Disketten für eine Neuinstallation von MS-DOS bereit oder
erstellen Sie eine MS-DOS-Bootdiskette, auf der die Programme \prog{fdisk},
\prog{format} und \prog{restore} enthalten sind. Diese Diskette benötigen Sie,
um nach dem Backup von Diskette zu booten
und von dort aus Ihre Festplatte neu zu partitionieren. 

Halten Sie außerdem {\it zwei formatierte Disketten}
im Format Ihres Bootlaufwerkes (das ist das Laufwerk A:) bereit.

Bei der Installation werden Sie die folgenden, später genauer beschriebenen Schritte
durchführen:
\begin{itemize}
\item Löschen aller alten Partitionen Ihrer Festplatte. (Mit dem DOS-Programm \prog{fdisk})
\item Anlegen einer neuen, kleinen MS-DOS Partition. (Mit dem DOS-Programm \prog{fdisk})
\item Anlegen einer neuen SWAP Partition\footnote{
In dieser Partition lagert das Betriebssystem Teile des Hauptspeichers aus,
die gerade nicht benötigt werden.} für LINUX. (Mit dem LINUX-Programm \prog{fdisk})
\item Anlegen einer neuen, großen LINUX Partition. (Mit dem LINUX-Programm \prog{fdisk})
\item Installation von LINUX auf der LINUX Partition.
\item Erstellen von Boot-Disketten, um mit diesen Disketten LINUX zu starten.
\end{itemize}

Bevor Sie mit der Installation beginnen noch eine Anmerkung. Es wird zwar immer
vom Industrie-Standard PC gesprochen, doch in der Realität ist jeder PC anders.
Mal ist es die Grafik-Karte, dann der Plattenkontroller, der Bus oder das BIOS
durch das sich ein Rechner vom Anderen unterscheidet. Wenn man dann noch sieht
wie viel verschiedene Software es gibt (MS-DOS, Novell DOS, OS-2, ...) und
wie unterschiedlich diese auf den Rechnern installiert sein kann (C:, D:, ...),
ist klar, daß wir nicht alle möglichen Konfigurationen berücksichtigen können.

Sollte Ihnen während der Installation eine Abweichung zu unserer Anleitung
auffallen, so kann dies natürlich an der unterschiedlichen Konfiguration der
Rechner liegen. Sie sollten dann selbstständig herausfinden, wie Sie weiter
vorgehen müssen oder bei uns anrufen (Tel. 02331/9874368). Es sollte aber 
anhand dieser Anleitung immer klar sein, wie Sie im Prinzip weiter vorgehen müssen.

\section{Installation von LINUX}
\subsection{Anmerkungen}

Speicherbedarf: die spezielle LINUX-Version, die Sie von uns bekommen haben, benötigt
ungefähr 60MB an reinen Daten und Programmen. Dazu kommen noch ungefähr 16MB an sogenanntem
Swap-Space. Sie müssen sich also darüber im Klaren sein, ungefähr 80 MB Ihres Plattenplatzes
für das LINUX-System zur Verfügung zu stellen! Dazu kommt der Platz, den
Ihre eigenen Dateien und Programme belegen werden.

Legen Sie außerdem schon jetzt zwei formatierte
Disketten für Ihr Bootlaufwerk bereit. Auf diese Disketten wird am Ende der Installation
der Linuxkern aufgespielt. Wenn Sie später LINUX benutzen wollen, müssen Sie den Rechner
mit einer dieser Disketten booten. Die zweite Diskette wird nur für eine Sicherheitskopie der Ersten
benötigt.

\subsection{Start eines kleinen LINUX für die Installation}
Legen Sie die LINUX Bootdiskette in das Laufwerk A: ein. Wir gehen hier davon aus, daß das
Bootlaufwerk (A:) ein 3~1/2 Zoll Laufwerk ist. Wenn Ihr Diskettenlaufwerk A: ein 5~1/4~Zoll
Laufwerk ist und Ihr Laufwerk B: ein 3~1/2 Zoll Laufwerk, dann legen Sie alle Disketten bis auf
die Bootdiskette in Laufwerk B:. Sie bekommen dann von uns eine spezielle Bootdiskette
für Ihr Laufwerk A:.

Starten Sie nun den Bootvorgang durch betätigen von \taste{Reset} oder der
Tastenkombination \taste{Ctrl~-~Alt~-~Del} oder durch Einschalten Ihres Rechners. 
Es wird nun ein kleines Linuxsystem in den Rechner geladen und ausgeführt. Dies dauert etwas.
Danach erscheint auf dem Bildschirm folgende Meldung:

\begin{verbatim}
Please remove the boot Kernel disk from your floppy drive,
insert a disk to be loaded into the ramdisk, and press
[enter] to continue.
\end{verbatim}

Besitzer deren Laufwerk A: ein 3~1/2~Zoll Laufwerk ist, legen nun die Diskette {\bf color144}
in das Laufwerk A: ein. Diejenigen mit einem 3~1/2~Zoll Laufwerk als Drive B: legen die
Diskette {\bf color144} in das Laufwerk B: !

Drücken Sie nun die Taste \taste{Return}, bzw. die Taste \taste{Enter}.

Ein minimales ROOT-Filesystem wird nun in die Ramdisk geladen. Dies dauert wieder eine Weile,
in der einige Meldungen ausgegeben werden. Als letzte Meldung erscheint die Aufforderung sich
anzumelden. Tun Sie dies, indem Sie \eing{root} als Login eingeben. (Als Login wird die
Eingabe des Benutzernames bezeichnet. In diesem Fall geben Sie \eing{root} als Namen ein. Dies
ist ein auf jedem UNIX-System voreingestellter Benutzer, der sog. Super-User. Dieser Benutzer
hat alle Rechte und darf jede nur mögliche Veränderung am System vornehmen.)

Danach erscheint am linken Rand der Eingabezeile das Zeichen 
\begin{verbatim}
#
\end{verbatim}

Dieses zeigt an, daß das Installations-LINUX bereit ist, Eingaben von Ihnen entgegenzunehmen.
Bevor Sie Kommandos eingeben können, müssen Sie jedoch erst das System auf Ihre deutsche
Tastatur umstellen. LINUX geht ansonsten davon aus, daß Sie eine amerikanische Tastatur
besitzen. 

\subsection{Einstellen der deutschen Tastatur}
Zu diesem Zweck geben Sie folgende Befehle ein, wobei Sie beachten müssen, daß
LINUX noch von einer amerikanischen Tastatur ausgeht, Sie also anstatt \eing{/} 
ein \eing{-}, anstatt \eing{-} ein \eing{ß}, anstatt \eing{<} ein \eing{;} und 
anstatt \eing{|} ein \eing{'} (Hochkomma) eingeben müssen. Außerdem sind die
Buchstaben \eing{y} und \eing{z} vertauscht.

\begin{verbatim}
# cd /usr/lib/kbd/keytables
# gzip -d <gr-lat1.map.gz | loadkeys
\end{verbatim}

Danach hat LINUX die deutsche Tastatur erkannt. 

\subsection{Partitionieren der Festplatte unter LINUX}
Die Installation beginnt mit der Einteilung Ihrer Festplatte in verschiedene Partitionen.
Zu diesem Zweck müssen Sie das LINUX-Programm \prog{fdisk} aufrufen. Sie tun dies, indem Sie
einfach \eing{fdisk} eingeben. Es erscheint folgender Text als Antwort:

\begin{verbatim}
# fdisk
Using /dev/hda as default device!
 
Command (m for help):
\end{verbatim}

Das \prog{fdisk} Programm von LINUX hat im Prinzip die gleiche 
Funktionalität wie das von DOS.
Geben Sie nun den Buchstaben \eing{p}\footnote{für print} gefolgt von \taste{Enter} ein.
Sie sehen die DOS-Partition,
die Sie mit dem DOS-Programm \prog{fdisk} angelegt haben: 
(Diese Angabe nennen wir {\em Patitionsinfo}).

\begin{verbatim}
Command (m for help): p
 
Disk /dev/hda: 16 heads, 38 sectors, 683 cylinders
Units = cylinders of 608 * 512 bytes
 
   Device Boot  Begin   Start     End  Blocks   Id  System
/dev/hda1   *       1       1      34   10317    1  DOS 12-bit FAT
\end{verbatim}

Die Angabe erfolgt hier in Blocks, d.h.\ unsere 10 MB große DOS-Partition besteht
aus 10317 Blocks.
Wir wollen nun zwei weitere Partitionen für LINUX anlegen,
und zwar eine sogennante SWAP-Partition,
in die LINUX nicht benötigte Teile seines Hauptspeichers ablegen kann,
und eine Partition in der das Filesystem
abgelegt ist. LINUX wird in dieses Filesystem alle Dateien schreiben,
mit denen Sie später arbeiten werden. 

\subsubsection{Einrichten der SWAP-Partition}
Wir beginnen mit der Einrichtung der SWAP-Partition. Zu diesem Zweck geben Sie im Programm 
\prog{fdisk}\footnote{Das Programm läuft ja immer noch.}
die folgenden Befehle ein\footnote{Die Tasten, die Sie drücken müssen, haben wir
durch unterstreichen im Bild gekennzeichnet.}:

\begin{verbatim}
Command (m for help): n
                      -
Command action
   e   extended
   p   primary partition (1-4)
p
-
Partition number (1-4): 2
                        -
First cylinder (35-683): 35
                         --
Last cylinder or +size or +sizeM or +sizeK (35-683): +16M
                                                     ----
\end{verbatim}

Sie haben nun nacheinander die Befehle \eing{n} (für new partition), 
\eing{2} (für zweite Partition), \eing{35} (für den ersten freien Zylinder) 
und \eing{+16M} (für 16 Megabyte Größe) eingeben. Bedenken Sie, daß
der erste freie Zylinder bei Ihrer Platte wahrscheinlich nicht der 35. ist, sondern der
nächste freie Zylinder nach der in {\em Partitionsinfo} in der Spalte End angezeigten Angabe!
Nur in unserem Fall (10 MB DOS-Partition) ist es Zylinder 35.
Die Größe der SWAP-Partition sollte 16 Megabyte nicht unterschreiten. Sonst können Sie
einige Programme, die viel Hauptspeicher brauchen, evtl. nicht starten.

Sie können sich die bisher eingestellten Partitionen jetzt anschauen,
indem sie nochmal \eing{p} eingeben:

\begin{verbatim}
Command (m for help): p
                      -
Disk /dev/hda: 16 heads, 38 sectors, 683 cylinders
Units = cylinders of 608 * 512 bytes
 
   Device Boot  Begin   Start     End  Blocks   Id  System
/dev/hda1   *       1       1      34   10317    1  DOS 12-bit FAT
/dev/hda2          35      35      88   16416   83  Linux native
\end{verbatim}

Wie Sie sehen, haben Sie eine weitere Partition angelegt. Diese wird jedoch noch nicht als
SWAP-Partition erkannt!

Um aus dieser allgemeinen LINUX-Partition eine SWAP-Partition zu machen, müssen Sie den
Id-Code noch ändern. Geben Sie dazu den
Buchstaben \eing{t} gefolgt von \taste{Enter} ein, danach geben Sie 
\eing{2} ein, danach \eing{L} gefolgt von einem \taste{Enter} und zuletzt geben 
Sie dann \eing{82} \taste{Enter} (dies ist der Code für die Swap-Partition von 
LINUX) ein. Es entsteht folgender Dialog:

\subsection{Einrichten eines Benutzers}
Damit Sie nicht immer als Benutzer \datei{root} arbeiten müssen, erstellen Sie
sich als erstes einen neuen Benutzer.
Geben Sie dazu den Befehl zum Einrichten eines neuen Users \eing{adduser} ein.

\begin{verbatim}
darkstar:~# adduser
\end{verbatim}

Sie werden nun gefragt, wie der neue Benutzer heißen soll. Geben Sie Ihren Nachnamen ein
(in diesem Beispiel werden wir den Namen ,,testuser'' benutzen):

\begin{verbatim}

\end{verbatim}

Das Programm \prog{adduser} gibt nun die Aufforderung aus, Vorname und Nachname einzugeben.
In diesem Beispiel soll der Testuser Hubert heissen.

\begin{verbatim}

\end{verbatim}

Als nächste Eingabe wird eine Nummer erwartet, die sogenannte ,,Group ID''. Bestätigen Sie
diese Eingabe einfach nur mit \taste{Return}. Das \prog{adduser}-Programm 
vergibt dann selbst die nächste freie Nummer.

\begin{verbatim}

\end{verbatim}

Der nächste Schritt ist die Eingabe der ,,User ID''. Geben Sie auch 
hier einfach nur \taste{Return} ein. \prog{adduser} wird die nächste freie

\begin{verbatim}

First unused uid is 501
UID [501]: 
\end{verbatim}

Danach stellt Ihnen \prog{adduser} die Frage an welcher Stelle Sie Ihr sog. ,,Home Directory''

\begin{verbatim} 
Home Directory [/home/testuser]: 
\end{verbatim}

\begin{verbatim}

\subsection{Kopieren der LINUX Bootdiskette}
Der nächste Schritt besteht nun darin, eine Kopie Ihrer Bootdiskette
anzufertigen. Dazu ist es notwendig, zuerst den Inhalt der Bootdiskette 
auf die Festplatte zu kopieren. Zu diesem Zweck geben Sie das folgende 
Kommando ein:

\begin{verbatim}
darkstar:~# dd if=/dev/fd0 of=bootdisk
\end{verbatim}

Diese Eingabe kopiert alle Daten der Diskette in Ihrem Bootlaufwerk in ein File mit dem
Namen \datei{bootdisk}. Dies dauert ein oder zwei Minuten. Danach nehmen Sie Ihre Bootdisk
aus dem Bootlaufwerk heraus und legen eine andere formatierte Diskette in das Bootlaufwerk ein.
Um das File mit dem Inhalt Ihrer Bootdiskette auf die neue Diskette zu kopieren, geben Sie
folgendes Kommando ein:

\begin{verbatim}
darkstar:~# dd if=bootdisk of=/dev/fd0
\end{verbatim}

In Ihrem Bootlaufwerk befindet sich danach eine Kopie der Bootdiskette. Testen Sie diese 
Diskette, indem Sie das nächste Mal LINUX von dieser Diskette booten.

\subsection{Anmelden als ,,normaler'' Benutzer}

einloggen. Zu diesem Zweck müssen Sie sich erstmal als User \eing{root}

Sie bekommen nun wieder den Login-Prompt. Geben Sie nun Ihren Loginnamen (Nachnamen) ein:
(In dieser Beispiel natürlich der Name \eing{testuser}). Danach müssen Sie noch das Passwort
eingeben (hier ebenfalls \eing{testuser}):

\begin{verbatim}
darkstar login: testuser
Password: 
Last login: Fri Mar 18 10:59:04 on console
Linux 0.99.15. (Posix).
darkstar:~$ 
\end{verbatim}

Das Passwort wird aus Sicherheitsgründen nicht angezeigt während Sie es eintippen.
Sie haben sich nun erfolgreich unter Ihrem Namen angemeldet und können z.B. 
das X-Windows System mit dem Kommando \prog{startx} starten.

\section{Ausschalten eines LINUX-Rechners}

Sie dürfen einen Rechner unter LINUX keinesfalls einfach ausschalten. Dies 
könnte zu Inkonsistenzen auf der Festplatte führen. Um den LINUX-Rechner 
auszuschalten, müssen Sie folgende Schritte ausführen:

\begin{itemize}
\item Melden Sie sich zuerst ab.
\item Melden Sie sich dann als User root an.
\item geben Sie das Kommando \eing{halt} ein.
\item warten Sie bis die Meldung:
\end{itemize}

\begin{verbatim}
halt: sending all processes the TERM signal ......................
halt: sending all processes the KILL signal ..
Unmounting file systems .....
Done
The system is halted
\end{verbatim}
auf dem Bildschirm erscheint. Danach können Sie den Computer ausschalten oder DOS
booten. Um DOS zu booten legen Sie einfach eine DOS Diskette in Laufwerk A: oder
legen Sie keine Diskette in Laufwerk A:. Wenn Sie mit LINUX arbeiten wollen,
legen Sie beim einschalten eine der erstellten LINUX-Bootdisketten in Laufwerk A:.

\section{Die Online-Hilfestellung}
LINUX stellt für alle Befehle eine Kurzbedienungsanleitung bereit, die sog. man-Page. Sie
können diese Kurzanleitungen jederzeit mit dem Kommando \prog{man} aufrufen. Sie
können z.B. Hilfestellung zum \prog{man}-Kommando bekommen, indem Sie
\eing{man man} eingeben.

Die man-Pages sind übrigens ins Deutsche übersetzt worden und im LINUX-Anwenderhandbuch
abgedruckt. 

Welche Kommandos unter einem UNIX-System verfügbar sind entnehmen Sie bitte der
Literatur über UNIX. Falls ein Kommando aus der Literatur nicht so arbeitet wie
dort beschrieben, so sehen Sie in den man pages nach wie dieses Kommando in unserem
System arbeitet.

Ansonsten können Sie auch mit dem Autor dieses Textes in Verbindung treten und
Ihre Fragen direkt stellen. Auch bei allen Problemen oder Abweichungen zu unserer
Anleitung sollten Sie sich bei uns melden. Sehr häufig können Sie die Lösung
aber auch selbst herausbekommen, indem Sie die Anleitungen aufmerksam durchlesen.
In der Praxis ist das der ''normale'' Weg.
